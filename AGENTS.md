# AGENTS

## Commit Discipline

- Create atomic commits.
- Commit often while making changes.
- Push after each commit that you create.
- Do not batch unrelated work into a single commit.
- Do not create empty commits.

## Starting Work

- If the repository already contains non-confidential uncommitted changes, stage and commit them before starting new work, then push that commit.
- If existing uncommitted material appears confidential, sensitive, or clearly not ready to preserve, do not commit it automatically.
- When existing changes are confidential or ambiguous, stop and ask before proceeding.

## Scope Control

- Keep each change focused on the current task.
- Do not revert unrelated user changes.
- Prefer product specs and plans inside this repository over repo-external notes.
